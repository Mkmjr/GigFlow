## Section 1 – Product Vision
Gigflow is a product based web application which serves as a platform for providing and posting freelance projects.
Gigflow helps client in posting the desired projects and their reasonable amount for completion of the project, whereas freelancers can pick up the desired project as per their capability and skills, earn money and refer.
Gigflow has been created as a portfolio project that showcases all the modern prodcut based team principle and architecture been used to build a project.

## Section 2 - Problem Statement
In the digital era and high usage of internet and social media, clients still finds it very hard to find a skill based matched or high potential freelancers for their gigs. Also with an increasing number of users, freelancers also finds it very hard to find clients for thier gigs and pay as per their expertise.
GigFlow addresses these challenges by providing a secure, scalable, and role-based web application that streamlines project posting, freelancer discovery, proposal management, authentication, and collaboration while maintaining a clean and intuitive user experience.

To resolve this issue, Gigflow a centralised web/mobile based platform is been built to overcome this issue for all the users of this application. The main users of Gigflow applications are:
1. Admin - Managing all the users, gigs, users accounts, spam accounts detection and access related issues.
2. Client - Posting and managing their posted gigs, hiring potential freelancers and accessing the application.
3. Freelancer - Applying for gigs, managing their applied gigs and delivering projects.
